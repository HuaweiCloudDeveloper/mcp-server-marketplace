简介

语音交互服务（Speech Interaction Service，简称SIS）是一种人机交互方式，用户通过实时访问和调用API（Application Programming Interface，应用程序编程接口）将语音识别成文字或者将文本转换成逼真的语音等。 MCP（Model Context Protocol）为智能体（Agentic）应用提供底层支撑，通过标准化接口与服务集成架构， 显著扩展Agent的能力边界与协作效率。该协议支持企业快速接入多类服务并实现统一调用，降低系统复杂度和开发成本，助力构建高性能、可扩展的企业级智能体解决方案。

前提条件

  • 云服务账号：准备华为云提供商的访问凭证
  • AK/SK准备：获取华为云提供商的Access Key和Secret Key
  • Python环境：Python 3.12+
  • UV工具：跨平台安装工具

安装UV

Linux/macOS：

curl -LsSf https://astral.sh/uv/install.sh | sh

Windows：

powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

使用方法

启动服务

# 克隆仓库
git clone git@github.com:HuaweiCloudDeveloper/sis-mcp-server.git

# 进入项目目录
cd /your-path/mcp-server-main

# 设置环境变量 
export HUAWEI_ACCESS_KEY="xxxx"
export HUAWEI_SECRET_KEY="xxxx"

# STDIO（默认）：最适合本地工具和命令行脚本
uv run mcp-server-sis --transport stdio --port 8888

# 可流式传输的 HTTP：推荐用于 Web 部署
uv run mcp-server-sis --transport http  --port 8888

# SSE：为了与现有的 SSE 客户端兼容。
uv run mcp-server-sis --transport sse --port 8888

快速开始

支持以下客户端与MCP服务交互：

  • Cherry Studio（官方客户端）
  • Cursor（官方客户端）
  • Windsurf（官方客户端）
  • VS Code + Cline

客户端通信配置示例

1. stdio 模式

{
  "mcpServers": {
    "stdio-sis": {
      "type": "stdio",
      "command": "uv",
      "args": [
        "run", 
        "/your-path/mcp-server-main/huaweicloud_services_server/mcp_server_sis/src/mcp_server_sis/run.py"
      ]
    }
  }
}

2. http 模式

{
  "mcpServers": {
    "http-sis": {
      "type": "http",
      "url": "http://localhost:8888/mcp",
    }
  }
}

3. sse 模式 - SSE 传输已弃用，可能会在未来版本中移除

{
  "mcpServers": {
    "sse-sis": {
      "type": "sse",
      "url": "http://localhost:8888/sse"
    }
  }
}

API工具列表

热词管理接口

  • DeleteVocabulary
    通过热词表id删除热词表。
  • UpdateVocabulary
    更新一个热词表,更新成功返回id。
  • ShowVocabulary
    通过热词表id查询热词表的信息和内容。
  • CreateVocabulary
    新建一个热词表,创建成功返回id。
  • ShowVocabularies
    查询用户所有热词表列表。

语音合成接口

  • RunTts
    语音合成,是一种将文本转换成逼真语音的服务。用户通过实时访问和调用API获取语音合成结果,将用户输入的文字合成为音频。通过音色选择、自定义音量、语速,为企业和个人提供个性化的发音服务。

语音识别接口

  • RecognizeShortAudio
    一句话识别接口,用于短语音的同步识别。一次性上传整个音频,响应中即返回识别结果。
  • CollectTranscriberJob
    该接口用于获取录音文件识别结果及识别状态。
  • PushTranscriberJobs
    录音文件识别。
  • RecognizeFlashAsr
    极速版ASR(Restful API 接口, 适用于音频(文件大小<=100M,语音时长<=30分钟)文件的同步识别。

以及更多用于元数据、可用区、磁盘管理和监控的API。
